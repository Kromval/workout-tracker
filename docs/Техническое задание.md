
## 1. Название проекта

**Workout Planner / Конструктор тренировок**

## 2. Цель проекта

Разработать простое веб-приложение для:

- просмотра каталога упражнений;
- создания и редактирования персональных тренировок;
- запуска тренировки с таймерами и звуковыми сигналами;
- ведения истории выполненных тренировок;
- отслеживания прогресса через календарь;
- хранения пользовательских данных локально в браузере.

## 3. Технологические ограничения

- Frontend: `HTML`, `CSS`, `JavaScript` без фреймворков.
- Хранение данных: `localStorage`.
- Источник встроенных упражнений: локальный `.json`.
- Работа приложения без обязательного сервера.
- Поддержка светлой и темной темы.
- Поддержка двух языков интерфейса: `ru` / `en`.

# 4. Основные сущности данных

## 4.1. Упражнение

Упражнение — элемент справочника.

### Поля упражнения:

- `id` — уникальный идентификатор;
- `name`:
    - `ru`
    - `en`
- `shortDescription`:
    - `ru`
    - `en`
- `instruction`:
    - `ru`
    - `en`
- `effect`:
    - `ru`
    - `en`
- `type`:
    - `ru`
    - `en`
    - пример: растяжка / кардио / силовое / мобилити / статика
- `muscles` — массив тегов мышц/частей тела на английском;
- `tags` — массив дополнительных тегов;
- `executionMode` — тип выполнения упражнения:
    - `reps` — по повторениям;
    - `time` — по времени;
    - `hold` — удержание;
    - `custom`
- `tempo` — объект таймингов выполнения, опционально;
- `estimatedCalories` — примерный расход калорий;
- `image` — ссылка на схему/картинку, опционально;
- `isCustom` — флаг пользовательского упражнения.

### Пример `tempo`

Для упражнений по повторениям:

{  
  "eccentric": 2.0,  
  "concentric": 0.5,  
  "pauseTop": 0.0,  
  "pauseBottom": 0.0  
}

Для статических или временных упражнений `tempo` может отсутствовать.

---

## 4.2. Тренировка

Тренировка — пользовательский шаблон набора упражнений.

### Поля тренировки:

- `id`
- `title`
- `description`
- `createdAt`
- `updatedAt`
- `isPreset` — предустановленная тренировка или пользовательская
- `items` — список элементов тренировки
- `defaultRestBetweenExercises`
- `themeColor` — опционально для карточки
- `tags` — опционально

---

## 4.3. Элемент тренировки

Элемент тренировки — конкретное использование упражнения в рамках шаблона тренировки.

### Поля:

- `id`
- `exerciseId`
- `sets`
- `reps` — если тип `reps`
- `durationSec` — если тип `time` или `hold`
- `distance` — опционально
- `restBetweenSetsSec`
- `restAfterExerciseSec`
- `tempoOverride` — переопределение темпа, опционально
- `notes` — заметка к упражнению в конкретной тренировке
- `order`

---

## 4.4. История выполнения тренировки

История нужна отдельно от шаблона тренировки.

### Поля:

- `id`
- `workoutId`
- `workoutTitleSnapshot`
- `startedAt`
- `endedAt`
- `durationSec`
- `status`:
    - `completed`
    - `aborted`
    - `interrupted`
- `completedItems` — данные о фактическом прохождении
- `note`
- `ratingEmoji`
- `estimatedCaloriesBurned`
- `totalExercisesCompleted`
- `totalSetsCompleted`

---

## 4.5. Пользовательские настройки

### Поля:

- `language` — `ru` / `en`
- `theme` — `light` / `dark` / `system`
- `soundEnabled`
- `volume`
- `customAudio` — набор загруженных пользовательских сигналов
- `favoriteExerciseIds`
- `calendarViewMode`
- `lastOpenedWorkoutId`

# 6. Функциональные требования

## 6.1. Главная страница

На главной странице отображаются:

- календарь прогресса;
- список тренировок пользователя;
- предустановленные популярные тренировки;
- быстрые действия:
    - создать тренировку;
    - открыть упражнения;
    - продолжить последнюю незавершенную тренировку, если есть.

### Для списка тренировок:

У каждой тренировки должны быть кнопки:

- открыть;
- редактировать;
- дублировать;
- удалить.

### Для календаря:

- отображение дней с выполненными тренировками;
- визуальная индикация интенсивности/наличия активности;
- клик по дню открывает список выполненных тренировок и заметки;
- отображение рейтинга тренировки через emoji.

---

## 6.2. Страница списка упражнений

Отображается каталог упражнений в виде карточек.

### Каждая карточка должна содержать:

- название;
- тип;
- мышцы/теги;
- краткое описание;
- кнопку “в избранное”;
- кнопку “открыть”.

### Должны быть доступны:

- поиск по названию;
- поиск по мышцам;
- поиск по типу;
- фильтры по выпадающим спискам:
    - тип;
    - мышцы;
- отображение избранных упражнений вверху списка;
- переключение между встроенными и пользовательскими упражнениями;
- создание собственного упражнения.

---

## 6.3. Страница создания упражнения

Пользователь может создать собственное упражнение.

### Пользователь может задать:

- название `ru/en`;
- описание `ru/en`;
- инструкцию `ru/en`;
- эффект `ru/en`;
- тип `ru/en`;
- мышцы;
- теги;
- тип выполнения;
- темп;
- калории;
- ссылку на картинку или загрузить путь к локальной картинке;
- сохранить упражнение.

### Ограничения:

- обязательные поля должны быть валидированы;
- название не должно быть пустым;
- числовые значения должны быть неотрицательными.

---

## 6.4. Страница просмотра упражнения

Отображаются полные данные упражнения:

- название;
- описание;
- инструкция выполнения;
- эффект;
- тип;
- мышцы;
- теги;
- темп;
- примерное сжигание калорий;
- изображение, если есть;
- кнопка “добавить в тренировку”;
- кнопка “в избранное”.

---

## 6.5. Страница создания тренировки

Пользователь может создать тренировку:

- с нуля;
- на основе популярного шаблона;
- копированием существующей тренировки.

### При создании тренировки доступны:

- ввод названия;
- ввод описания;
- добавление упражнений из каталога;
- настройка параметров для каждого упражнения;
- drag and drop для изменения порядка;
- удаление элемента из тренировки;
- установка отдыха между подходами;
- установка отдыха между упражнениями;
- задание заметок к конкретному элементу;
- сохранение тренировки.

### Дополнительно:

- должна быть поддержка упражнений:
    - по повторениям;
    - по времени;
    - на удержание.

---

## 6.6. Страница редактирования тренировки

Функционально аналогична созданию, но работает с существующей тренировкой.

### Дополнительно:

- автосохранение черновика;
- подтверждение перед выходом без сохранения;
- возможность быстро продублировать тренировку.

---

## 6.7. Страница описания тренировки

Должна отображать:

- название;
- описание;
- список упражнений по порядку;
- параметры каждого упражнения;
- общую оценку длительности;
- примерный расход калорий;
- кнопки:
    - начать тренировку;
    - редактировать;
    - удалить;
    - дублировать.

---

# 7. Функционал проведения тренировки

## 7.1. Общий сценарий

Пользователь запускает тренировку.  
Упражнения выполняются в порядке, заданном пользователем.

Система должна:

- показывать текущее упражнение;
- показывать текущий подход;
- показывать текущий повтор или время;
- показывать следующее упражнение;
- показывать общий прогресс тренировки;
- показывать общий таймер тренировки;
- показывать крупный таймер текущего этапа.

---

## 7.2. Таймеры и логика этапов

Приложение должно поддерживать:

- выполнение повторений по темпу;
- выполнение упражнений по времени;
- интервалы отдыха между подходами;
- отдых между упражнениями.

### Для упражнений по темпу:

На основе `tempo` должны рассчитываться этапы:

- старт повторения;
- опускание;
- нижняя пауза;
- подъем;
- верхняя пауза.

---

## 7.3. Звуковая индикация

Во время тренировки должна быть звуковая индикация.

### Базовые требования:

- стандартный тихий тик для секунд/интервалов;
- отдельный акцентированный сигнал для смены фазы движения;
- отдельный сигнал для старта упражнения;
- отдельный сигнал для начала отдыха;
- отдельный сигнал для завершения отдыха;
- отдельный сигнал для завершения упражнения;
- отдельный сигнал для завершения тренировки.

### Пользовательские сигналы:

Пользователь может загрузить собственные аудиофайлы для отдельных событий.

### Требования к кастомным сигналам:

- поддержка популярных аудиоформатов (`mp3`, `wav`, `ogg`);
- предварительное прослушивание;
- привязка сигнала к событию;
- fallback на стандартные сигналы, если файл недоступен.

---

## 7.4. Управление тренировкой

Во время выполнения должны быть доступны кнопки:

- пауза;
- продолжить;
- пропустить текущее упражнение;
- `+10 сек`;
- `-10 сек`;
- завершить тренировку досрочно.

### Дополнительно:

- подтверждение перед досрочным завершением;
- защита от случайного двойного клика;
- корректная работа после паузы.

---

## 7.5. Завершение тренировки

После завершения или прерывания пользователь может:

- сохранить результат;
- добавить заметку;
- поставить рейтинг по emoji от 1 до 5.

### Шкала рейтинга:

1. `😐`
2. `🙂`
3. `😅`
4. `😵`
5. `🤮`

Да, шкала у тебя слегка токсичная, но зато честная.

---

## 7.6. Восстановление незавершенной тренировки

Если пользователь закрыл вкладку, перезагрузил страницу или случайно вышел:

- текущая сессия должна быть сохранена;
- при следующем входе приложение должно предложить восстановить тренировку.

---

# 8. История и прогресс

## 8.1. Календарь прогресса

Календарь должен:

- отображать выполнение тренировок по датам;
- позволять открывать детали дня;
- показывать количество тренировок за день;
- отображать рейтинг и заметку;
- различать завершенные и прерванные тренировки.

## 8.2. Базовая статистика

На главной странице или отдельным блоком должна отображаться базовая статистика:

- количество тренировок за неделю;
- количество тренировок за месяц;
- общее время тренировок;
- общее количество выполненных упражнений;
- текущая серия активных дней.

# 9. Дополнительный функционал

## 9.1. Избранное

Пользователь может добавлять упражнения в избранное.  
Избранные упражнения отображаются в верхней части списка.

## 9.2. Популярные тренировки

В приложении должны быть предустановлены несколько популярных тренировок, например:

- общая разминка;
- домашняя силовая тренировка;
- кардио на 15 минут;
- растяжка;
- пресс/кор.

## 9.3. Импорт и экспорт

Пользователь должен иметь возможность:

- экспортировать свои данные в JSON;
- импортировать данные из JSON;
- сбросить все локальные данные с подтверждением.

Экспортируются:

- пользовательские упражнения;
- тренировки;
- история;
- настройки;
- избранное.

## 9.4. Локализация

Должна быть возможность смены языка интерфейса `ru/en`.

### Требования:

- текст интерфейса локализуется;
- названия/описания упражнений берутся из соответствующего языка;
- если у пользовательского упражнения нет второй локализации, допускается fallback на основной язык.

## 9.5. Темы оформления

Поддержка:

- светлой темы;
- темной темы;
- автоматического режима по системной теме — опционально.

# 10. Нефункциональные требования

## 10.1. Производительность

- приложение должно быстро загружаться;
- интерфейс должен корректно работать без заметных задержек при типичном объеме данных;
- поиск и фильтрация упражнений должны работать в реальном времени.

## 10.2. Надежность

- ошибки в `localStorage` должны обрабатываться;
- приложение не должно падать при поврежденных данных;
- при невозможности прочитать пользовательские данные должен использоваться безопасный fallback.

## 10.3. Удобство использования

- интерфейс должен быть понятным без обучения;
- ключевые кнопки должны быть заметны;
- на мобильных устройствах базовый интерфейс должен оставаться рабочим.

## 10.4. Валидация

- пустые обязательные поля запрещены;
- отрицательные значения запрещены;
- тренировка не может быть сохранена без упражнений;
- пользователь должен подтверждать удаление тренировки/упражнения/данных.

# 15. Критерии приемки

Приложение считается соответствующим ТЗ, если:

1. Пользователь может просмотреть встроенные упражнения.
2. Пользователь может создать свое упражнение.
3. Пользователь может создать несколько тренировок.
4. Пользователь может редактировать, дублировать и удалять тренировки.
5. Пользователь может запускать тренировку и проходить ее по шагам.
6. Во время тренировки корректно работают:
    - таймер;
    - сигналы;
    - пауза;
    - пропуск;
    - изменение времени.
7. После завершения пользователь может сохранить заметку и рейтинг.
8. Результат отображается в календаре.
